## Fixed

- Ship `eslint-import-resolver-typescript` and `prettier-plugin-tailwindcss` as runtime `dependencies`. Both are referenced statically by the default `preferences` preset, so consumers were crashing at lint time with `Cannot find module` when the package was installed without dev dependencies.

## Added

- Local git hooks (husky + lint-staged) covering pre-commit auto-fix (prettier, eslint on TS and `package.json`, sort-package-json, markdownlint), a `Co-Authored-By: Claude` commit-message trailer strip, and a direct-push-to-`main` block. Each pre-commit task is scoped to staged files only. Also adds `sort-package-json` as an explicit devDep so the hook works offline, and ignores `CHANGELOG.md` in prettier so the hook doesn't ping-pong against markdownlint.
- `actionlint` and `yamllint` validation for `.github/workflows/*.{yml,yaml}` and YAML across the repo. Wired into both the pre-commit hook (best-effort: skips with a `brew install …` hint when the tool isn't on `PATH`; semantic errors block commits, warnings don't) and a new parallel `yaml-lint` CI job (always enforced; both tools cached, pinned to `actionlint v1.7.5` and `yamllint 1.37.1`). Adds `lint:yaml` and `lint:workflows` scripts plus `.yamllint.yml` at the repo root.
- "When to use" JSDoc headers on every rule preset (`rules/*.ts`) and inline JSDoc on every opt-in re-export in `index.ts`. The JSDoc rides through to `dist/**/*.d.ts`, so consumer IDE hover tooltips on imports like `{ sanity }` now show the guidance.

## Changed

- Bump `globals` to 17.6.0.
- Skip the `CI / 🔬 Build & Lint` and `Claude Code Review` workflows on `changeset-release/*` PRs (the auto-generated "release: version packages" PR), which added no value on a generated version bump and were burning CI minutes plus Claude credits.
